# v1.1 Backlog — Pro Tier Polish

Tickets opened from observations during the first end-to-end Pro tier demo run on 2026-04-28 (session `969260a4-e48e-412a-a8d2-acda7b29ab20`, tx `5vq7d1rC...EQDMj`). The Pro tier is shipped and demonstrable — these are reliability + observability improvements, none are blockers.

---

## P1 — Reliability

### V11-01 — Bound embedding concurrency + retry with backoff
- **Owner**: data-engineer
- **Why**: First Pro run failed mid-ingestion on `text-embedding-3-small` TPM (`1M tokens/min`). Second run succeeded but only because we picked a narrower idea. A repeatable demo cannot rely on prompt selection.
- **What**: in `packages/gecko-core/src/gecko_core/ingestion/embed.py` (or wherever `openai.embeddings.create` is called), wrap calls with:
  - `asyncio.Semaphore(8)` to cap concurrency at 8 in-flight embedding calls
  - tenacity-style retry on `RateLimitError`, exponential backoff `1s → 2s → 4s → 8s`, max 4 attempts
  - log retry attempts at INFO so CloudWatch shows the throttle visibly
- **Acceptance**: replay the original "habit tracker that pays in stablecoins" idea on Pro; should complete despite the 15+ source pull.
- **Why not bigger model**: `text-embedding-3-large` is 3× cost, hits the same TPM ceiling under load, and erodes margin from 88% → 84%. Concurrency cap fixes the real problem.

### V11-02 — Pro tier retry-without-recharge
- **Owner**: software-engineer
- **Why**: when a Pro session fails after x402 settles (e.g. our embedding rate-limit case), the user is out $0.75 with no result. Currently no retry path; a re-run charges fresh.
- **What**: when `/research/pro` 202s and the session subsequently fails before `set_result`, the API should:
  - Mark the session `status='failed'` with the error in `result_json.error`
  - Issue a one-time `retry_token` (signed, 24h TTL, scoped to session_id) returned via the `events_url` SSE stream's terminal event
  - Add `POST /research/pro/{session_id}/retry?token=...` that re-runs `_run_pro_debate` against the same session_id, no new payment, requires the retry_token
  - MCP client surfaces the retry on failure: "Pro session failed (rate limit). Retry without recharge? [y/n]"
- **Acceptance**: induce an embedding rate limit during a Pro session; receive the retry_token; retry succeeds 60s later; on-chain tx and economics show single $0.75 charge across both attempts.

### V11-03 — XMLParsedAsHTMLWarning silenced + correct parser dispatch
- **Owner**: software-engineer
- **Why**: cosmetic CloudWatch noise during ingestion when an RSS/XML feed is parsed with `html.parser`. Confuses log review.
- **What**: in `packages/gecko-core/src/gecko_core/ingestion/web.py:295`, branch on response Content-Type:
  - `application/xml`, `application/rss+xml`, `application/atom+xml`, or `text/xml` → `BeautifulSoup(body, "lxml-xml")`
  - everything else → existing `BeautifulSoup(body, "html.parser")`
- Add `lxml>=4.9` to `gecko-core` deps if not already present.
- **Acceptance**: rerun ingestion against a known RSS feed (e.g. `https://newsletter.pragmaticengineer.com/feed`); zero `XMLParsedAsHTMLWarning` lines in CloudWatch.

## P2 — Observability

### V11-04 — AG2 token counting from ClawRouter responses
- **Owner**: software-engineer
- **Why**: `transcript.turns[*].tokens_in` and `tokens_out` are all 0 in the first Pro session. AG2's `a_generate_reply` returns response shapes that bypass our token-counting wrapper (`_extract_token_counts` in `pro/__init__.py`). Means per-agent COGS attribution in `session_costs.agent` is currently uninformed.
- **What**: in `pro/__init__.py`, after each `a_generate_reply` call, fetch the underlying `ChatCompletion` usage from the AG2 agent's last reply context. ClawRouter passes through OpenAI's `usage.prompt_tokens` / `completion_tokens` — they're on `agent.client.last_response` or similar. Pin the exact attribute by capturing one full AG2 message-with-usage and committing the path.
- **Acceptance**: rerun a Pro session; `transcript.total_tokens_in > 0`; per-turn `tokens_in/out` match what `gecko-mcp economics <id>` shows in the ledger.

### V11-05 — Per-agent cost rows actually populated
- **Owner**: data-engineer
- **Why**: dependent on V11-04. Once token counts arrive, `_run_pro_debate` should call `store.append_session_cost(line_item="llm", agent=event.agent, cost_usd=...)` per turn. Verify it does.
- **What**: confirm the `on_event` callback wired in `workflows.research()` actually inserts a row per `turn_end`. Run `SELECT agent, count(*), sum(cost_usd) FROM session_costs WHERE session_id=$1 GROUP BY agent` against the Pro session id; expect 5 rows (one per agent) with non-zero costs.
- **Acceptance**: `gecko-mcp economics <pro_session_id>` shows 5 LLM line items, one per agent, summing close to the embedding+orchestration COGS.

## P3 — UX polish

### V11-06 — `gecko-mcp --version` flag
- **Owner**: software-engineer
- **Why**: during the smoke I ran `gecko-mcp --version` and got "No such option". `gecko-mcp doctor` is the closest thing. Standard CLI convention says `--version` should work.
- **What**: add Click `@click.version_option(package_name="gecko-mcp")` to the root CLI in `packages/gecko-mcp/src/gecko_mcp/cli.py`. Same for `gecko-core` if it has a CLI.
- **Acceptance**: `gecko-mcp --version` prints `gecko-mcp, version 0.1.1` and exits 0.

### V11-07 — `install.sh` notes uv cache pinning
- **Owner**: software-engineer
- **Why**: during local smoke, `uv tool install gecko-mcp` resolved an old cached `gecko-core 0.1.0` alongside `gecko-mcp 0.1.1`. End users hitting this would see import errors when trying Pro tier.
- **What**: in `gecko-claude/install.sh`, change `uv tool install --force gecko-mcp` to `uv tool install --force --reinstall-package gecko-core gecko-mcp` so the dep is always pulled fresh.
- **Acceptance**: on a machine with `gecko-core 0.1.0` cached, the installer ends with `gecko-core 0.1.1` resolved.

## What's NOT in this backlog

- Migration to `text-embedding-3-large` — rejected. 3× cost, doesn't fix concurrency, erodes margin.
- Routing embeddings through ClawRouter — rejected. ClawRouter is LLM-completion proxy, embeddings are different traffic class. Adds infra without fixing the rate limit.
- Caching embeddings across sessions — out of scope for v1.1; revisit when we have multi-tenant load and dedupe-able idea space.
- Mainnet x402 cutover — separate workstream, post-Shipathon.
- Privy per-project wallet isolation — v2 roadmap.
