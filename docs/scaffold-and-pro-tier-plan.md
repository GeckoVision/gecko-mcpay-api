## Architecture: two repos + one server addition

### Repo 1 — `gecko-mcpay-api` (this repo)
Stays as-is. Server-side. Adds:
- **AutoGen Pro tier** (`packages/gecko-core/orchestration/pro.py`) — multi-agent GroupChat for `tier="pro"`. Each specialist is a `RoutedAssistant` whose LLM client points at ClawRouter (or OpenAI direct as fallback).
- **Tool registry** (`packages/gecko-core/tools/`) — server-side wrappers for any paid tool the Pro orchestrator might need (Whisper, Exa, Browserbase, Perplexity). Each wrapper internally calls frames.ag `/x402/fetch` against the *server's* wallet (we've prepaid a frames.ag wallet for server-side tool spend, separate from user payments).
- **Per-session cost telemetry** — already shipped tonight; extend to log per-agent + per-tool costs.

### Phase B.5 — Per-project vaults (1-2 days, can parallel with Phase B)

**Architectural goal:** each Gecko project gets its own frames.ag sub-wallet so spend on one idea can't drain another, and per-project budgets are first-class.

**Reality check from the web3-engineer:** frames.ag's default tier limits users to **1 wallet per chain**. Multi-wallet only opens at Silver (1) / Gold (5) / Diamond (unlimited). **Most free-tier users can't get a second wallet without referrals or a paid tier.** That makes the design "architecturally sound but operationally degraded" for the funnel head.

**v1 strategy: lazy provisioning with graceful fallback**

1. `gecko-scaffold init` writes `<project>/.gecko/project.json` with `{name, project_id (uuid), wallet_address: null}`.
2. First paid call (e.g. `gecko_research`) checks `project.json`:
   - If `wallet_address` is null → inspect tier (`GET /wallets/{u}/wallets`)
   - If `counts.solana < limits.solana` → `POST` create with `idempotencyKey = sha256(username + project_id)`, persist address
   - **Else** → fall back to main wallet, log `"project budget isolation unavailable on your tier"`, set `wallet_address = "__main__"` so we don't retry on every call
3. Mid-session `insufficient_funds` does NOT auto-fall-back to the main wallet (defeats the budget). Surface verbatim, prompt `gecko project fund`.

**Soft sandbox for free-tier users (the fallback):** use frames.ag's policy controls (`max_per_tx_usd`, `daily_limit_usd`) to bound per-project spend within the main wallet. Loses cryptographic isolation, keeps budget discipline.

**Schema (one new table, additive on `sessions`):**

```sql
-- infra/supabase/migrations/<ts>_projects.sql
CREATE TABLE projects (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  username text NOT NULL,
  name text NOT NULL,
  wallet_address text,           -- null until provisioned (or "__main__")
  wallet_id text,
  budget_usd numeric,            -- soft cap, client-side enforced
  created_at timestamptz NOT NULL DEFAULT now(),
  UNIQUE (username, name)
);
ALTER TABLE sessions
  ADD COLUMN project_id uuid REFERENCES projects(id),
  ADD COLUMN paid_from_wallet_address text;  -- audit, denormalized
```

**API change to `FramesAGWallet`:**

```python
async def x402_fetch(
    self, url: str, *,
    method: str = "POST",
    body: dict | None = None,
    max_payment_usd: str | None = None,
    wallet_address: str | None = None,   # NEW — routes to specific sub-wallet
    max_attempts: int = 3,
) -> dict[str, Any]:
    ...
    if wallet_address is not None:
        payload["walletAddress"] = wallet_address
    ...
```

Plus two new helper methods on `FramesAGWallet`: `list_wallets()`, `create_wallet(chain_type, idempotency_key)`.

The provisioning decision logic (when to create vs fall back) lives in a new module `packages/gecko-core/src/gecko_core/payments/project_vault.py`, not in `FramesAGWallet` — keeps frames.ag transport clean.

**Client-side per-project file** at `<project>/.gecko/project.json`:
```json
{
  "project_id": "uuid-v4",
  "name": "bonito-hotel-guide",
  "frames_username": "ernani",
  "wallet_address": "...",
  "wallet_id": "...",
  "created_at": "2026-04-27T..."
}
```

Forkable (no secrets — apiToken stays in `~/.agentwallet/config.json`). Optional global index `~/.gecko/projects.json` for `gecko project list`.

**Race conditions:** flock on `project.json` (handles 99% of CI parallelism) + frames.ag-side `idempotencyKey`. Idempotency semantics on `POST /wallets` aren't documented — flagged as ping-frames item below.
