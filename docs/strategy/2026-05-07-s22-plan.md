---

## 1. Context

S20 built the categorized-knowledge spine (taxonomy, Mongo schema, vertical/category axes). S21 reframed the pioneer story and consolidated pay.sh integration thinking. We have the wedge, we have the storage, we have the manifest understanding. What we do not have is **a single deliverable a stranger can run on their machine and walk away believing in.** Stellar37 + Colosseum deadline pressure means S22's job is artifact, not foundation.

Why neobank: it's the densest (vertical, category) cell pay.sh actually serves — KYC, card-issuing, BaaS, fraud, on-chain settlement are all directly purchasable as x402 endpoints today. Neobank lets us tell the full pay.sh-as-source story with five real paid calls instead of synthetic ones. Other verticals (DEX, marketplace, prediction market) are deferred to S23+ — proving the loop once is more valuable than half-proving it three times.

Why MCP-first: per CLAUDE.md, the MCP server is the frames-style surface for Claude Code. Wiring the example app's `.mcp.json` at the local Gecko MCP closes the consumption loop the way builders will actually use Gecko in the wild. Direct HTTP calls would be a different product story; this sprint commits to the MCP one.

## 2. Goals (measurable)

- **G1 — Corpus density:** ≥200 neobank-tagged chunks live in Mongo Atlas, with provider diversity ≥3 distinct `KnowledgeSource` values across `paysh_manifest`, `paysh_live`, and `web`. Verified by aggregation query in §8.
- **G2 — Runnable example:** `examples/neobank-on-gecko/` in the public `gecko-claude` repo runs end-to-end. Developer clones → `npm install` → `claude code` opens with Gecko MCP wired in `.mcp.json` → `gecko_research --idea "build a Solana neobank with USDC"` returns ≥5 cited chunks where ≥1 citation came from a `paysh_*` source.
- **G3 — Demo video:** 5-minute screencast (Loom or OBS) published to `gecko-claude` README and posted on Twitter. Founder opens Claude Code → asks Gecko for neobank build context → MCP returns categorized chunks → founder ships scaffold → cited build artifact appears.

## 5. Falsifiers

Three concrete signals that would kill or pivot the sprint mid-flight:

1. **Pay.sh manifest is structurally unusable.** If on Day 1–2 the pay.sh `index.json` shape rejects normalization (missing skill descriptions, no stable IDs, or auth-walled) or fewer than 20 of 75 entries can be parsed into useful discovery chunks, the `paysh_manifest` source angle dies. **Pivot:** drop N2 + the manifest half of N5; double down on `web` source neobank seeds and reframe G1's diversity bar as `paysh_live + web + tavily`.

2. **Live x402 calls cost more than the budget cap to produce useful chunks.** If the 5 chosen pay.sh providers each cost >$1 per useful response or return responses too thin to chunk meaningfully, N4/N5 economics break. **Pivot:** record one live call per provider (5 total, ≤$5 spend) as a fixture, ship the example app against the fixture, and call out the live-mode toggle as future work in N9 video.

3. **MCP integration in the example app is fragile across machines.** If on Day 4 the fresh-clone smoke fails (path issues, env-var leaks, MCP handshake instability) and the fix is non-trivial, G2 is at risk. **Pivot:** ship a Docker-based example (`docker compose up`) instead of bare `npm install`. Eats one day but trades reliability for surface elegance.

## 7. Risks (top 3)

1. **Pay.sh provider response variability blows up the classifier.** Live x402 responses are heterogeneous (some are markdown, some JSON, some structured-blob). The S20 classifier was tuned on web + tavily prose. **Mitigation:** N5 normalizes provider responses through a thin per-source adapter before classification; treat the adapter layer as part of N4's acceptance, not an N5 afterthought.
2. **Demo video re-shoot loop.** Recording a clean 5-min walkthrough often takes 5–10 takes. Day 6 alone is thin. **Mitigation:** N9 starts script + first take on Day 5 evening; Day 6 is for editing + publish, not first-take recording.
3. **Stellar37/Colosseum judges may want HTTP, not MCP.** The locked decision is MCP-first. If submission feedback during the sprint signals judges expect a hosted demo URL, we'd need to bolt on a hosted variant. **Mitigation:** N9 video doubles as the hosted demo surrogate (judges watch the artifact). If they require live URL, post-sprint S23 ticket — do not pivot mid-sprint.
