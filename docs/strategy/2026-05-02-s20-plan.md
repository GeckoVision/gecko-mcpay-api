## 4. Risks & gotchas

1. **Adding KILL is a Verdict-enum change.** Schema-drift test must catch it (Pattern A). Migration on the verdict-vocab change must update `tests/eval/baselines/*.json` to use the new label or document the legacy mapping. Breaking change for any external SDK consumer reading `Verdict`.
2. **Per-session reranker batching changes the rerank input set.** Today each question reranks against its own retrieval slate. After batching, multiple questions share one slate. R3's "no semantic win" result was on per-question reranking — batching may *change* the answer. Re-run the chunk-level A/B (`scripts/voyage_chunk_ab.py`) post-#3 with batched semantics; if the gate trips after batching, that's the time to flip the default, not before.
3. **VERDICT-URL-IMPL-01 reads from `gecko_cache.judge_transcripts`** (Mongo, not Supabase). Confirm hash field is indexed before the route ships — verdicts could be O(n) lookups otherwise.
4. **#9 RAG-EVAL-LABELS-01 needs a real, populated Mongo session for label grounding.** R3 used `6cc0a982...` (x402/agentic-payments). Hand-labeling against that single corpus risks topic-overfitting. Either re-run dogfood ingest on 5 distinct topics first, or accept the topic-bias caveat and document it.
5. **Cross-agent prompt drift.** Lead 3 (#1) lands while #2, #5, #6, #7, #8 modify prompts. Order #1 BEFORE the prompt-modifying tickets so the consistency test guards the changes.
