## 3. Risks & gotchas

1. **Voyage AI is preview.** No production SLA. R3 must include a "fall back to no-rerank" code path that triggers on Voyage 5xx or >2s latency. Don't make the demo dependent on a third-party preview API being up.

2. **Reranker latency budget.** Demo runbook target is <90s end-to-end. Mongo hybrid is ~200–400ms, synthesizer is the bulk. A reranker call per question adds 300–800ms (Voyage `rerank-2`). If multi-question synthesis fans out 5 reranker calls, that's 4s — which the runbook will feel. R2 should batch reranker calls per session, not per question.

3. **Fresh-corpus eval baseline drift.** S18 cuts over to a fresh Mongo corpus. The S17 holdout-live numbers were measured on the Supabase corpus. D3 (`S19-EVAL-GAP-01`) reset the baseline before R3 measures the rerank delta — otherwise the A/B confounds storage-cutover effects with rerank effects.

4. **Tradeable-surface scope creep.** S1 is spec-only on purpose. If "let's just sketch a working URL" creeps in, it pulls verdict-hash-storage, paywall middleware, and x402 settlement into S19 — none of which fit. Keep it a doc.

5. **`gecko-claude` repo doesn't exist on disk.** Per S17 plan notes. D2 (`install.sh`) is gated on whoever owns static hosting at `app.geckovision.tech`. Verify upload path before claiming D2 is done.

6. **Supabase chunk drop is irreversible.** S3 only runs if cutover has been clean for ≥3 days. If S18 surfaced any latent Mongo issue post-sprint, defer S3 to S20.
