## 3. Risks & gotchas

1. **v1_block tear-out is S19 territory but the pro-debate prompt depends on its "always-render-heading" contract.** S18-V1BLOCK-AUDIT-01 (#10) is the recon ticket so we don't sleepwalk into a regression in S19. Do NOT remove v1_block in S18.

2. **Tradeable-judgment requires settlement, and x402 facilitator is stub-only for now.** S18-TRADEABLE-SURFACE-01 is spec-only on purpose. Live settlement for verdict purchase is web3-eng territory and a 2-week slice minimum. Don't let demo enthusiasm pull live settlement into S18.

3. **`gecko-claude` repo doesn't exist on disk.** Per S17 plan notes. `app.geckovision.tech/skill.md` is live (verified) but `install.sh` execution is gated on whoever owns the static hosting. S18-INSTALL-DISTRIB-01 must verify the upload path, not just the script.

4. **Verdict-hash determinism is harder than it looks.** Corpus fingerprinting must be order-stable; embed cache hits change retrieval order subtly. S18-VERDICT-HASH-01 (#9) needs to hash the *retrieved* citation set after rerank, not the corpus pre-rank, otherwise the hash is non-deterministic across reruns.

5. **Eval threshold drift after S17 hybrid reranker is unknown.** S18-EVAL-GAP-01 deferred to S19; the slim cutover eval in M6 stands in.

6. **Hybrid-search parity is the long pole.** Postgres `match_chunks_hybrid` (86-line SQL) does pgvector + tsvector + RRF in one query. Mongo equivalent is `$vectorSearch` + `$search` + app-side RRF. Tuning until citation order matches within Â±1 position on fixture queries is M4's blocker. **Pattern C applies:** contract-test against recorded fixtures before flipping the flag.

7. **Atlas grant runway.** $400 credits valid through August 2026 â‰ˆ 50 months at $8/mo or 6 months at $57/mo (M10 dedicated). Plan assumes shared/serverless tier is sufficient for solo + small-beta load. If pre-launch traffic forces M10, runway shortens â€” flag for `business-manager` review.

8. **No re-embed during cutover.** Embedding model stays OpenAI `text-embedding-3-small` (1536-d). Voyage AI swap is explicitly S19 (`project_voyage_ai_s19_followup`). Don't co-schedule a model swap with a storage swap.

9. **Mongo MCP tools speed up M1/M4 â€” use them.** `mcp__plugin_mongodb_mongodb__*` is now installed. Use `list-collections`, `collection-indexes`, `collection-schema`, `aggregate`, and `explain` for live inspection during dev rather than booting a separate Atlas UI. The `mongodb:mongodb-search-and-ai` skill is the prerequisite read for M4 â€” Hybrid Search + Vector Search index design has gotchas the docs don't surface.
