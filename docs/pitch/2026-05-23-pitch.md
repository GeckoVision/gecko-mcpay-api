## 1. The problem

Autonomous agents now move real money on-chain. But they **act without judgment** — they pull data + skills and execute. No second opinion. No grounding. No memory of *why*. No way to know if the agent is confidently hallucinating.

> "Treating a wallet like an API call breaks the moment the agent is autonomous."

The result: **you cannot trust an agent with real capital.** Today you either babysit it or pray.

## 2. Why now

The agent economy is exploding — machine-to-machine value, OKX OnchainOS, ElizaOS, x402 metering. Everyone is building agents that **act**. Almost nobody is building the layer that decides whether they **should**. Execution is commoditized. **Judgment is the gap.**

## 3. What Gecko is

A **verifiable judgment oracle.** You — or your agent — propose a trade or strategy. Gecko's multi-voice adversarial panel returns a grounded verdict:

> **SAFE / DEFER / REJECT** — with investor-canon **citations** and **surviving dissent you can see.**

It plugs in *above* any agent, via x402 pay-per-call. The agent executes; Gecko decides if it's a good idea first.

## 4. The wedge — why it's defensible (and provable today)

**The measured gating delta.** We backtest on our *own* multi-regime data: the trades the oracle calls SAFE **beat** the ones it calls DEFER — confidence-interval-clean, across trend-up, trend-down, transitional, and chop. Current locked figure: **+0.6% per trade, sign-robust, N=78**, hardening as we extend the regime coverage.

We don't ask you to *trust* the oracle. **We show you the number.**

> This is the escape from the "you can't prove it without users / can't get users without proof" trap that kills most judgment products: we prove it on historical data we own, before anyone risks a cent. Our own adversarial panel raised exactly that objection — and this is the answer.

## 5. Why we're different

| They | What they do | Why we're not them |
|---|---|---|
| **Generic AI trading agents** (the crowd) | *Act* on a strategy, no judgment | We're the **referee**, not a racer. We don't compete to be the best bot — we tell any bot when it's wrong. |
| **Security/policy layers** (e.g. AgentARC) | "Is this transaction *safe / allowed*?" — deterministic policy + simulation | We answer "is this a *good decision*?" — grounded judgment + alpha + dissent. **Different axis. Complementary, not competing.** |
| **Data feeds** (CoinGecko, Cambrian) | Sell *data* per call | We sell *judgment grounded in* data. Data is cheap; the verdict is the value. |

**The moat:** the measured gating delta + the grounded verdict corpus + visible, auditable dissent. Orchestration is table stakes — we don't lead with it.

## 6. How it works (the architecture — integrations are plumbing)

The **Gecko Oracle** is the product, at the center. Everything around it is swappable infrastructure — *like saying we use AWS or Supabase*:

- **Execution:** OKX OnchainOS agentic wallet (one sub-wallet per strategy)
- **Privacy/security:** Cloak.ag (integration + Solana connection partner)
- **Yield / vault:** Kamino
- **Chain:** Solana

The reference trading agent is a **proof artifact**, not the product. The oracle is what we sell; the rest is how it reaches the chain.

**Three modes of trust:** `paper` → `local` (ElizaOS, your machine) → `hosted safe mode` (sandboxed env + Cloak privacy + allocation policies).

## 7. Honest by design (the brand)

Gecko's whole reason to exist is that LLM agents hallucinate confidence. Our answer is an oracle that **surfaces dissent instead of hiding it.** We dogfood it: we ran our own pitch through the oracle, and it told us where we were weak. An oracle you can audit is the antidote to a black box.

## 8. Who it's for (ICP)

> **Builders and desks running autonomous agents on Solana (ElizaOS / OnchainOS) who need to trust agents with real capital** — plus risk-conscious on-chain traders who want a grounded second opinion before they commit.

*(This is the named-ICP gap to close with one paid pilot / LOI — the single highest-value validation step.)*

## 9. Business model

**x402 pay-per-call** — judgment as a metered service. Cache-then-charge keeps it economical (re-verdict on triggers/cadence, never per-tick — so latency is a non-issue). Reference price band in the agent-service market: $0.001–$0.2 / call.

## 10. Proof points (traction)

- A **validation spine** that gates every claim — block-bootstrap CIs, leakage traps, walk-forward, default-REJECT acceptance gates.

- A **live paper reference agent** (the proof artifact) + **x402 + MCP + Claude-skill distribution** (composable with the Solana ecosystem).
